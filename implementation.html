Building our platform from ground up

Implementation

Implementing the design in Sharepoint was a challenging process as our team were not familiar with the user interface of the design page, and the multiple power apps that were available to us. However, due to the extensive research and prototyping that had been done, we began to learn the ropes of the Sharepoint interface and after a number of weeks, majority of the functionalities were implemented as intended.

Once a set number of platforms have been identified, the next steps were to make a note of each platform, categorising the functionalities and overall design of their system, looking at the pros and cons, hence providing us with more information on how we would like to build our project proposal engine.

These will be outlined below.

Key Features

 

Submission Portal and display of projects

 
Sharepoint
The submission portal was implemented using the exisiting functionality of Microsoft Forms. The form was separated into sections for a more user-friendly experience. The form is connected to a list within the SharePoint site, which will automatically add the data of the new responses to the list using Microsoft Power Automate.

The data is then presented catagorically in a Microsoft List which can then be filtered, and queried with the inbuilt features within Microsoft Lists. Data in the fields of are visually represented using pills, and a range of different views based on the need of the reviewer.

Microsoft Power Automate flow for project proposals

The image above shows the flow of data in the submission of projects. When a project is submitted, this acts as a trigger for this flow which obtains the response details and adds it to the appropriate project submissions list with the correct data and adds it to the excel sheet storing the project proposals.

The submission portal in django was implemented using inbuilt django forms which accesses the fields required from the project model. When a submitter submits a project for approva, this sends a POST request to the backend server of django which stores the data appropriately into the Project model. This data is then displayed in a table in the project dashboard ordered by submission date, by iterating through the stored instances in the Project.

With the use of the django-filters libraries, an extensive query can be performed based on multiple fields which is implemented in the projects grid and lists which can query projects based on specified criteria by the user so projects can be categorised according to preference.

The taggable manager library is used to parse tags with the use of javascript and bootstrap so tags can be associated with projects.

Reviewing Projects and giving feedback

Sharepoint

Each user can be assigned to a project using the inbuilt Microsoft 365 feature in SharePoint, where they can then give feedback by clicking a button which will trigger a flow that will prompt the user for inputs such as the feedback they wish to give, the RAG rating, and a dropdown multi list option of extra feedback details. These feedback details will then be appended to the feedback columns in the proposal submissions list, and the number of red/amber/green ratings will be incremented based on the reviewer's answer.

This will then be used to update the specific project in the SharePoint list as well as the excel sheet to reflect the newly added changes.

The list column is formatted using json to only show when a project has not been accepted / rejected and clicking on the button will trigger the flow, and only members within the site organisation who can edit the site are able to trigger the flow.

Approval of projects

Sharepoint

Another button is displayed in the sharepoint list, if the status of the project is neither accepted nor rejected. This button can be triggered by any reviewer once all feedback has been sent, to send an email to the approver of the project, giving details of the project, the feedback, the RAG ratings and list of reviewers. This starts a Microsoft Approval process, where the approver can accept or reject the project through that and this will update the status accordingly based on the answer. If the project is accepted, a new item will be created in the Accepted Projects list and the submitter will be informed via email.

The approvers can be selected in the flow, and this can be a group of leads as well as one lead, and preferences can be set such as the ability to ensure all approvers respond, or the ability for the system to update the status based on the response of the first approver.

Storage of data - Databases

Sharepoint

The lists can be exported with a query or without a query as a csv file by clicking the export button that is built into the SharePoint lists. The accepted projects and project proposals are also stored in an excel sheet, stored in the document library of the SharePoint site. The feedback data is also stored in the excel sheets, which can be mined by companies to show feedback from different individuals.
