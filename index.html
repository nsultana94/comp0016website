IXN1 CFP Portal Engine for Project Proposals

Portal Engine for efficient collection, reviewal and distribution of projects across many companies and universities.

Project Abstract

This project is run along with corporate clients. It develops an open-source web portal for the IXN subscriber companies to deploy across their businesses, allowing project proposals to be advertised, collected and reviewed.

It should allow an organisation to collect project proposals using existing IXN templates, and feature a submission panel where reviewers can review projects with comments and a Green-Amber-Red style.

Projects should be allowed to be shared amongst universities and past completed projects should be indexed and archived.

Solution

We developed a system design based on the implementation using SharePoint, due to its robust inbuilt network and as office 365 is an intranet most companies and universities use. Allows fluid data collection and distribution across companies and universities and uses many pre-existing features.

With Sharepoint, we were able to develop a fluid way of data collection and using Microsoft Lists, an efficient way to display and organise the collected project proposals and efficient technique of reviewing and giving feedback to these projects using Microsoft Power Automate.

On the other hand, we will be using Django for front-end and back-end for a public version of the portal engine, requested by the NHS. Their systems are not reliant on Microsoft and therefore would prefer a standalone software.
