IXN1 CFP Portal Engine for Project Proposals

Portal Engine Tool for efficient collection, reviewal and distribution of projects across many companies and universities.

Project Abstract


This project is run alongside corporate clients. Its final result is an open-source web portal for the IXN subscriber companies to deploy across their businesses, allowing project proposals to be advertised, collected and reviewed.

It will allow an organisation to collect project proposals using existing IXN templates and submission forms, and feature a submission panel where reviewers can review projects with comments and a Green-Amber-Red style marking system.

Projects should be allowed to be shared amongst universities and past completed projects should be indexed and archived.


Solution


This portal engine tool is influenced mainly by the existing IXN scheme managed by University College London. Currently, project proposals are collected using Microsoft Forms so this engine allows an efficient collection of these projects as well as efficient distribution, reviewal and management of the project data.

Our aim and solution were to develop two separate platforms, the first one using the Microsoft products available via the wide gamut of Microsoft 365.

We implemented this system design based on an innovative approach using Microsoft SharePoint, due to its robust inbuilt network and office 365 being an intranet most companies and universities use. In adddition, SharePoint allows fluid data collection and distribution across companies and universities and uses many pre-existing features.

With Sharepoint, we were able to utilise Microsoft Lists as a means of data collection and Microsoft Power Automate as an efficient way to display and organise the collected project proposals. Using Microsoft Flow, we were able to create flows within the system that allowed us to develop an effective and creative technique of reviewing and giving feedback to these projects.

On the other hand, we developed an open-source, public version of the portal engine for the NHS client, using the Python-based web framework, Django, for its back-end and Bootstrap 4 along with its incorporation of JavaScript, CSS, HTML for the front-end. Their systems were not reliant on Microsoft and therefore preferred a standalone software, which is more trustworthy within the clinical environments.


Achievement and Impact


Overall, we delivered 98% of our functional requirements and 90% of our optional requirements. Both systems have been deployed for use and are successfully up and running on our clients' servers. The SharePoint system will hopefully be tested and used for the current IXN scheme and for future projects, by integrating the system with the other IXN system projects currently under development. We have been in contact with Great Ormond Street Hospital for possible use of our platform for collecting projects, as well as distribution within the NHS for project proposals internally. We expect the platform to come into usage for our clients soon and to target a wide network of potential users across the UK and not only.

  • Our SharePoint system will hopefully be used for collection of future IXN projects and to integrate it within the other groups working on the IXN workflow, such as integrating it with the front-end portal team and the students at University of Bristol.

  • This can be deployed across many internal organisations for use. They can integrate the system within their own existing site by importing the site flows and site design.

  • Microsoft will also hopefully use this for collection of projects within their internal student programme.

  • Our Django system could be used to collect projects from internal hospitals and trusts to collect project ideas regarding software engineering and could be integrated into an idea portal to collect project ideas and distribute them accordingly.

  • Django is safe to use on most operating systems and most browsers. It uses a SQLite 3 database which is very stable and inbuilt modules and dependencies which are easily installable for other systems.
